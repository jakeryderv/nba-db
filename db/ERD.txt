RELATIONSHIPS
================================================================================

  +----------+         +----------+         +--------------------+
  |  teams   |---------<  games   >---------|  player_game_stats |
  +----------+    1:M  +----------+   1:M   +--------------------+
       |                    |                        ^
       |                    |                        |
       | 1:M                | 1:M                    | M:1
       |                    |                        |
       v                    v                   +----------+
  +------------------+  +------------------+    | players  |
  | team_game_stats  |  | team_game_stats  |    +----------+
  +------------------+  +------------------+

  Relationship Details:
  ---------------------
  teams (1) ----< (M) games.home_team_id     One team has many home games
  teams (1) ----< (M) games.away_team_id     One team has many away games
  teams (1) ----< (M) team_game_stats        One team has many game stat records
  teams (1) ----< (M) player_game_stats      One team has many player performances

  players (1) --< (M) player_game_stats      One player has many game stat records

  games (1) ----< (M) team_game_stats        One game has exactly 2 team stats
  games (1) ----< (M) player_game_stats      One game has many player stats
  games (1) ----< (M) shot_attempts          One game has many shot attempts
  players (1) --< (M) shot_attempts          One player has many shot attempts
  teams (1) ----< (M) shot_attempts          One team has many shot attempts


CONSTRAINTS
================================================================================

  Primary Keys:
  -------------
  seasons.id              VARCHAR(10)    e.g., '2024-25'
  teams.id                BIGINT         NBA team ID
  players.id              BIGINT         NBA player ID
  games.id                VARCHAR(20)    NBA game ID
  team_game_stats.id      INT            Auto-increment
  player_game_stats.id    INT            Auto-increment
  shot_attempts            (game_id, event_id)

  Foreign Keys:
  -------------
  games.home_team_id          -> teams.id
  games.away_team_id          -> teams.id
  team_game_stats.game_id     -> games.id
  team_game_stats.team_id     -> teams.id
  player_game_stats.game_id   -> games.id
  player_game_stats.player_id -> players.id
  player_game_stats.team_id   -> teams.id
  shot_attempts.game_id       -> games.id
  shot_attempts.player_id     -> players.id
  shot_attempts.team_id       -> teams.id

  Unique Constraints:
  -------------------
  teams.abbreviation                      3-letter team code
  team_game_stats (game_id, team_id)      One stat record per team per game
  player_game_stats (game_id, player_id)  One stat record per player per game
  shot_attempts (game_id, event_id)        One record per official game event

  Check Constraints:
  ------------------
  games:
    - home_score >= 0
    - away_score >= 0
    - home_team_id != away_team_id

  player_game_stats / team_game_stats:
    - points >= 0, rebounds >= 0, assists >= 0
    - fgm >= 0, fga >= 0, fgm <= fga
    - fg3m >= 0, fg3a >= 0, fg3m <= fg3a
    - ftm >= 0, fta >= 0, ftm <= fta

  teams:
    - year_founded BETWEEN 1946 AND 2030

  seasons:
    - start_year >= 1946
    - end_year = start_year + 1
    - games_count >= 0, players_count >= 0

  shot_attempts:
    - valid period, clock, distance, and coordinate ranges
    - shot_type is a 2PT or 3PT field goal

NORMALIZATION (3NF)
================================================================================

  1NF - First Normal Form:
    - All attributes contain atomic values (no arrays or repeating groups)
    - Each record is uniquely identifiable via primary key
    - All tables have defined primary keys

  2NF - Second Normal Form:
    - Meets 1NF requirements
    - All non-key attributes fully depend on the primary key
    - No partial dependencies (stats depend on full game+player/team combo)

  3NF - Third Normal Form:
    - Meets 2NF requirements
    - No transitive dependencies
    - Team details only in teams table (not repeated in games)
    - Player details only in players table (not repeated in stats)

  Intentional Denormalization:
    - 'season' column exists in games, team_game_stats, player_game_stats, and shot_attempts
    - Rationale: Avoids expensive joins when filtering by season
