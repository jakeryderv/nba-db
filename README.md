# NBA Database

A PostgreSQL database system for NBA statistics with a Flask web interface.

## Tech Stack


- **Languages:** Python, SQL, HTML, CSS
- **Database:** PostgreSQL 15+
- **Backend:** Flask, Jinja2
- **Libraries:** psycopg2, pandas, nba_api
- **Styling:** Tailwind CSS (CDN)
- **Containerization:** Docker, Docker Compose
- **Tooling:** uv, pg_dump, psql

## Setup

### Prerequisites

- PostgreSQL 15+ (or Docker)
- Python 3.10+
- uv

### Installation

```bash
git clone <repo>
cd nba-db
uv sync

```

### Database Setup

**Option A: Local PostgreSQL**

```bash
./db/scripts/reset.sh nba_db dev
```

**Option B: Docker**

```bash
docker compose up -d

```

### Load Real Data

```bash
python etl/fetch.py
python etl/transform.py
python etl/load.py
```

### Run Application

```bash
flask run
```

## Usage

### Database Scripts

```bash
./db/scripts/build.sh    # Create DB + schema
./db/scripts/reset.sh    # Rebuild + seed
./db/scripts/test.sh     # Run all tests
./db/scripts/export.sh   # pg_dump to db/dump.sql
```

### Run Queries

```bash
psql -d nba_db -f db/queries/standings.sql
psql -d nba_db -f db/queries/stat_leaders.sql
```


## Database Schema

**Tables:** teams, players, seasons, rosters, games, player_game_stats, team_standings

**Key Features:**
- 3NF normalized schema
- CHECK constraints for data validation
- Triggers for automatic standings updates
- Stored procedures with error handling
- Indexed for query optimization
- Views for common reports


## Testing

```bash
./db/scripts/test.sh
```

Tests cover:
- Constraint validation (invalid data rejected)
- Trigger behavior (standings auto-update)
- Procedure error handling
